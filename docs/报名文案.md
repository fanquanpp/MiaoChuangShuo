## 正文

### 一、创意名称 + 创意介绍

**创意名称**: 喵创说

**想解决什么问题**:
独立与业余长篇创作者长期面临一个困境——云写作工具强制联网且存在数据隐私风险, 通用文本编辑器又缺少剧情线、人物关系、伏笔追踪等长篇专属能力。百万字级长篇创作过程中, 剧情分支容易混乱、人物关系难以对齐、伏笔挖坑不回收, 创作流程被迫碎片化分散在多个软件之间。

**为什么会想到做这个**:
自己作为一个有长期写作习惯的爱好者, 曾在断网环境下被云工具锁死无法继续创作, 也曾因通用编辑器缺少伏笔管理导致长篇作品挖坑不填。更关键的痛点是, 市面上的专业创作软件普遍采用订阅制, 业余创作者难以承担长期成本。于是想做一个完全本地化、零门槛、专为长篇叙事工作流设计的桌面工作站, 把编辑、剧情、人物、设定、伏笔五大模块集成在一起, 让创作回归专注本身。

**大概是什么产品**:
喵创说是一款基于 Tauri 2.0 + React 18 + Rust 构建的 Windows 桌面端离线长篇创作工作站, 安装包 10MB 以内, 完全本地运行, 无需账号登录, 数据 100% 存储于本地。

### 二、目标用户及痛点

**面向哪些用户**:

- 独立网文作者 (日更数千字, 需管理多条剧情线与大量伏笔)
- 业余小说爱好者 (周末创作, 难以承担专业软件订阅费用)
- 长篇连载作者 (需要长期维护人物关系与世界观一致性)
- 隐私敏感的创作者 (拒绝将未发表作品上传至云端)
- 断网环境创作者 (出差、高铁、咖啡馆等不稳定网络场景)
- 写作学习者 (需要结构化的长篇创作引导)

**在什么场景下使用**:
长篇连载创作、剧情结构梳理、人物关系可视化、伏笔全局追踪、世界观设定管理、断网环境专注写作。用户打开应用即可进入完整创作工作流, 无需在多个软件之间切换。

**当前痛点**:

- 云写作工具断网即失写作能力, 创作中断无法继续
- 云端存储存在隐私泄露风险, 创作者对未发表作品失去控制权
- 通用文本编辑器缺少长篇专属功能 (剧情线/人物关系/伏笔追踪)
- 专业创作软件订阅费用高, 业余写作者难以承担长期成本
- 百万字文档在通用编辑器中渲染卡顿, 滚动延迟影响创作节奏
- Electron 类应用安装包臃肿, 内存占用过大影响系统流畅

### 三、价值与意义

**效率提升角度**:
传统写作工具将创作者精力分散在文档、Excel、思维导图等多个软件之间。喵创说将编辑、剧情、人物、设定、伏笔五大模块集成于单一工作站, 配合 Ctrl+K 命令面板与 VS Code 风格快捷键体系, 创作效率显著提升。Rust 后端承载文件 IO、DAG 校验、字数统计等重型计算, 前端保持 60fps 流畅度, 百万字长文档滚动不卡顿。安装包体积相比 Electron 方案降低约 80%, 内存占用大幅减少。

**社会价值角度 (社会公益附加赛题立意)**:
喵创说坚持完全免费提供给独立与业余创作者群体, 服务那些无法承担云工具订阅费用的创作者。坚持完全本地化, 数据主权 100% 归创作者所有, 不收集任何用户数据, 不上传任何创作内容。这降低了长篇创作的工具门槛, 让更多有故事的人能够拿起键盘开始创作, 为内容创作生态贡献一份力量。让创作工具回归创作者手中, 是本项目的核心公益立意。
