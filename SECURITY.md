# 安全政策

## 报告安全漏洞

如果你发现喵创说存在安全漏洞，请**不要**通过公开 Issue 提交，而是通过以下渠道私下报告：

- **GitHub 私有安全公告（推荐）**：在仓库主页选择 `Security` - `Report a vulnerability`，填写漏洞详情
- GitHub Issue 中标注 `security` 标签私下联系维护者（请勿在公开 Issue 中包含可利用细节）

请在报告中包含以下信息：

1. 漏洞的详细描述与影响范围
2. 复现步骤（如涉及特定项目文件，可附上最小化样例）
3. 受影响的版本号
4. 你建议的修复方向（可选）

### 响应时间

- **确认收到**：3 个工作日内回复
- **初步评估**：7 个工作日内反馈是否接受为安全漏洞
- **修复发布**：严重漏洞 30 天内发布修复版本，一般漏洞随下一版本发布

### 报告原则

- 请勿在漏洞修复前公开披露细节
- 请勿利用漏洞获取他人数据或破坏系统
- 我们不会对符合本政策的善意报告者采取法律行动

## AI Key 安全说明

喵创说采用 BYOK（Bring Your Own Key）模式，用户自带 OpenAI 兼容 API Key。以下是关于 Key 安全的说明：

### 存储机制

| 平台 | 存储方式 | 安全等级 |
|------|---------|---------|
| Windows | DPAPI 加密后存储于 `%APPDATA%\MiaoChuangShuo\ai_config.json` | 用户级加密，仅当前 Windows 账户可解密 |
| macOS | keychain 系统钥匙串 | 系统级加密 |
| Linux | keyring 系统密钥环 | 系统级加密 |

### 安全承诺

- API Key **不会**以明文形式持久化到磁盘
- API Key **不会**通过网络发送至任何第三方中转服务器
- AI 调用采用 SSE 流式直连 LLM 服务，仅经过用户配置的 Base URL
- 旧版本使用的 Base64 编码存储方式已自动迁移为 DPAPI 加密

### 安全建议

1. **定期轮换 Key**：建议每 3 个月在 LLM 服务商后台轮换 API Key
2. **设置使用上限**：在 LLM 服务商后台为 Key 设置月度消费上限
3. **共享设备注意**：在共享设备上使用后，建议在「设置 - AI」中清除 Key
4. **不要分享配置文件**：`ai_config.json` 即使经过加密，也不应分享给他人

### 已知限制

- DPAPI 加密绑定 Windows 用户账户，**无法跨账户解密**（同一台机器切换用户后 Key 失效）
- 项目不收集任何遥测数据，因此**无法**在 Key 泄漏时主动通知用户

## 数据存储位置

| 数据类型 | 位置 | 说明 |
|---------|------|------|
| 应用配置 | `%APPDATA%\MiaoChuangShuo\` | 跨项目共享的用户偏好 |
| AI 配置 | `%APPDATA%\MiaoChuangShuo\ai_config.json` | DPAPI 加密 |
| 项目数据 | 用户自选目录 | 完全由用户控制 |
| 日志 | `%APPDATA%\MiaoChuangShuo\logs\` | 通过「设置 - 关于 - 打开日志目录」访问 |
